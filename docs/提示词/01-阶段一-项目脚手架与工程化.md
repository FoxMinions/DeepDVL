# 01 · 阶段一：项目脚手架与工程化

## 本阶段目标

用 Vite 起一个 **Vue 3 + TypeScript** 工程，配好 ESLint / Prettier / Stylelint，让 `npm run dev / build / lint / format` 全部可用且全绿。本阶段结束时，你有一个空页面的规范前端工程——还没有大屏内容，但「地基」打好了。

## 前置依赖

- 已初始化 Git 仓库（当前 DeepDVL 仓库已具备 README/LICENSE/.gitignore）。
- 本地有 Node.js（建议 LTS）与 npm。

## 为什么这么做

**为什么不直接开始画大屏？** 因为大屏项目代码量不小，没有工程化约束会很快失控：格式不统一、类型不严、lint 不过。先把「代码质量基础设施」立起来，后续每阶段才能稳。

**为什么选这套工具**：
- **Vite**：现代前端标配，启动快、配置简单，教学友好。
- **TypeScript strict**：大屏数据结构复杂（指标、趋势、排名、雷达…），类型能在编译期挡掉一大半低级错误。
- **ESLint flat config（v9+）**：新版配置更扁平、更易维护，是未来方向。
- **Prettier + Stylelint 分工**：Prettier 管格式（空格/换行），Stylelint 管样式规范（属性顺序/重复选择器），各司其职不冲突。

**教学要点**：理解 `package.json` 里 `scripts` 的设计——`dev`/`build`/`preview` 是 Vite 自带三件套，`lint`/`format`/`test`/`test:e2e` 是我们补的工程化命令。一个「能交付」的项目，这些脚本都得齐。

## 要落地的产物

- `package.json`：项目名 `deepdvl`，`type: module`，scripts 含 `dev / build / preview / lint / format`。
- Vite + Vue 3 + TS 工程文件：`index.html`、`vite.config.ts`、`tsconfig.json`、`tsconfig.app.json`、`tsconfig.node.json`。
- 入口：`src/app/main.ts`、`src/app/App.vue`（App.vue 先放一个占位标题「DeepDVL · 深度数界实验室」）。
- Lint 配置：`eslint.config.ts`（flat config，含 vue 与 ts 规则）、`.prettierrc.json`、`.stylelintrc.json`、`.prettierignore`。
- `src/app/main.ts` 挂载到 `#app`。
- 技术栈版本下限见 [00-项目总览](./00-项目总览与使用说明.md#二技术栈)。

> 本阶段**暂不**安装 ECharts/Pinia/Axios/MSW/Vitest/Playwright——这些在后续阶段按需引入，避免一开始装一堆用不上的依赖。

## 执行提示词

```
你是一个资深前端工程师。

请帮我用 Vite 初始化一个 Vue 3 + TypeScript 项目，项目英文名 DeepDVL，项目中文名「深度数界实验室」。请直接执行，不要只给方案。

要求：
1. 用 Vite 创建 Vue 3 + TS 工程，目录结构按下面来（不要用默认的 src/main.ts 平铺）：
   src/
     app/
       App.vue        # 占位：显示标题「DeepDVL · 深度数界实验室」
       main.ts        # 挂载到 #app
2. TypeScript 开启 strict 模式。
3. 配置 ESLint（v9+ flat config，eslint.config.ts）、Prettier（.prettierrc.json）、Stylelint（.stylelintrc.json + .prettierignore），三者不冲突。
4. package.json 的 scripts 必须包含：dev、build、preview、lint、format。
   - lint：eslint . && stylelint "src/**/*.{css,vue}"
   - format：prettier --write .
5. 本阶段只装脚手架相关依赖，不要安装 echarts/pinia/axios/msw/vitest/playwright（后续阶段再加）。
6. App.vue 先放一个居中的占位标题「DeepDVL · 深度数界实验室」，深色背景，不做复杂样式。
7. 版本下限：vue ^3.5、vite ^6、typescript ^5.6、eslint ^9、prettier ^3、stylelint ^16。装到更新大版本可保留，但须保证下方验证全绿。
8. 不要修改仓库已有的 README.md、LICENSE、.gitignore（如需补充忽略规则，在 .gitignore 末尾追加，不要删原有内容）。

验证（必须全部通过）：
- npm run build   # 含 vue-tsc，类型与构建都过
- npm run lint    # 无 error
- npm run dev     # 启动后页面显示占位标题，控制台无报错

完成后告诉我：本地路径、启动命令、浏览器地址、已安装的关键依赖与版本、已运行的验证命令及结果。
```

## 验收清单

- [ ] `npm run dev` 能启动，页面显示「DeepDVL · 深度数界实验室」
- [ ] `npm run build` 通过（含 `vue-tsc` 类型检查）
- [ ] `npm run lint` 无 error
- [ ] `npm run format` 能正常运行
- [ ] `src/app/` 目录结构正确（非平铺 `src/main.ts`）
- [ ] `tsconfig` 开启了 strict
- [ ] 仓库原有 README/LICENSE/.gitignore 未被破坏
- [ ] 浏览器控制台无报错

## 思考题 / 进阶

1. 为什么 `package.json` 要写 `"type": "module"`？去掉会怎样？
2. ESLint 的 flat config 和老的 `.eslintrc` 有什么区别？为什么 v9 默认前者？
3. Prettier 和 ESLint 都能管代码风格，为什么不只用一个？（提示：职责边界）
4. 进阶：试着给 `lint` 脚本加上 `--max-warnings 0`，让 warning 也阻断——想想教学项目里这么做利弊各是什么。
