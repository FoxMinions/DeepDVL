# 00 · 项目总览与使用说明

> 本文档是 DeepDVL 系列提示词的**总纲**。开始任何一个阶段前，请先读完本文档。

## 一、项目定位

**DeepDVL · 深度数界实验室** 是一个公开开源的实战教学项目，带你 **从 0 到 1 亲手制作一个数据可视化大屏**。

它不是「给你一个成品」，而是把大屏开发拆成 9 个阶段，每个阶段：

- 配一份**小提示词**（可直接喂给 AI 帮你实现）；
- 配一段**「为什么这么做」**的讲解（讲清设计理由，不只是照抄）；
- 配一份**验收清单**（做完自己勾选）。

## 二、技术栈

全部沿用主流 Vue 生态，与社区教学资源对齐，便于对照学习。

| 分类 | 技术 | 版本下限 | 用途 |
|---|---|---|---|
| 框架 | Vue 3 | `^3.5` | 视图层（`<script setup>` + Composition API） |
| 构建 | Vite | `^6` | 开发服务器与打包 |
| 语言 | TypeScript | `^5.6` | 类型安全（开启 strict） |
| 可视化 | ECharts | `^5.5` | 图表渲染 |
| 状态 | Pinia | `^2.2` | 全局数据状态 |
| 请求 | Axios | `^1.7` | API 请求（预留，默认走 mock） |
| Mock | MSW | `^2.4` | 拦截请求返回模拟数据 |
| 单测 | Vitest | `^2` | 单元测试 |
| E2E | Playwright | `^1.45` | 端到端浏览器测试 |
| Lint | ESLint | `^9`（flat config） | JS/TS/Vue 代码规范 |
| 格式化 | Prettier | `^3` | 代码格式化 |
| 样式 Lint | Stylelint | `^16` | CSS/Vue 样式规范 |

> **版本说明**：以上是**下限**。实际执行时若 AI 装到了更新的大版本（如 Vite 7/8、TS 6.x），可保留，但须保证 `npm run build` 与 `npm run lint` 全绿。教学项目锁版本下限是为了避免 AI 用过旧组合导致复现踩坑。

## 三、视觉基线：深空数据宇宙

> 这是贯穿全项目的视觉规范。各阶段提示词凡涉及样式，都以本基线为准。

### 设计意图

「深度数界实验室」—— 用**深空**的意象承载**数据**：星云是背景，星轨是装饰，节点是数据，流光是连接。整体克制、高级、清晰，重数据轻装饰，适合教学截图与开源展示。

### 色板（量化，直接用）

| 用途 | 变量名 | 值 | 说明 |
|---|---|---|---|
| 深空底 | `--bg` | `#030b18` | 最底层背景 |
| 靛紫层 | `--bg-indigo` | `#0a1230` | 渐变第二层 |
| 面板底 | `--panel` | `rgba(8,20,46,0.82)` | 面板半透明底 |
| 面板描边 | `--border` | `rgba(69,217,255,0.28)` | 青蓝描边 |
| 主强调 | `--cyan` | `#45d9ff` | 青蓝，主视觉色 |
| 次强调 | `--ice` | `#7be2ff` | 冰蓝，辅助 |
| 点缀 | `--indigo` | `#6f7bff` | 靛紫，少量点缀 |
| 良好 | `--green` | `#56f0c0` | 状态：良好 |
| 警告 | `--yellow` | `#f4c96b` | 状态：警告 |
| 危险 | `--red` | `#ff6b7a` | 状态：危险 |
| 主文字 | `--text` | `#eaf9ff` | 正文 |
| 次文字 | `--muted` | `#91b3ca` | 次级/说明 |

### 背景层叠（全局 body）

- 底色：`linear-gradient(145deg, #061431 0%, #030b18 48%, #061323 100%)`
- 星云：两层 radial-gradient，青蓝 `rgba(69,217,255,0.18)` 与靛紫 `rgba(111,123,255,0.12)`，散布在左上/右上
- 网格：`linear-gradient` 横竖线 `rgba(99,170,255,0.05)`，间距 `36px`，用 `mask-image` 顶部淡出

### 装饰元素（克制使用）

- **HUD 面板**：四角发光直角边框（青蓝），`box-shadow: inset 0 0 28px rgba(47,117,255,0.10), 0 0 24px rgba(69,217,255,0.18)`
- **流光线条**：`lines` 系列半透明青蓝虚线，做星轨/连接
- **脉冲节点**：`effectScatter`，带 `ripple` 涟漪
- **发光**：关键数字/节点用 `text-shadow` / `shadow-blur` 青蓝辉光

### 明确不做

- ❌ **不引入地图**（不引 china-map-geojson，不用 ECharts `map`/`geo`）。中心图用 `cartesian2d` 抽象坐标实现「数据宇宙」拓扑，避免地图数据依赖。
- ❌ 不做纯赛博风（不堆霓虹/扫描线/故障感）。
- ❌ 不做古风元素（与「深空」主题无关）。
- ❌ 装饰不得压过数据——任何装饰层若遮挡可读数据，一律降透明度或删除。

## 七、使用方法

1. **按顺序做**：从阶段 1 开始，做完一阶段 commit 一次，再进下一阶段。
2. **每阶段的「执行提示词」一节**：复制给 AI（Claude Code / Cursor / Copilot Chat 等），让它在你本地实现。
3. **AI 做完后**：按该阶段「验收清单」逐项检查，全勾再进下一阶段。
4. **「为什么这么做」一节**：建议做完后读一遍，理解每个设计决策——这是本项目的教学核心，别跳过。
5. **「思考题」一节**：留给你自己动手扩展，不强制，但做了会理解更深。

## 八、几条贯穿全程的纪律

1. **不把所有代码堆一个文件**——按目录结构分层。
2. **组件不直接读 mock**——一律走 `services` 层。
3. **样式用 CSS 变量**——颜色都从 `:root` 取，不硬编码，方便统一换肤。
4. **TypeScript strict**——类型要正确，`npm run build`（含 `vue-tsc`）必须过。
5. **每阶段都验证**——别攒着一堆最后一起测，问题会很难定位。
6. **不破坏上一阶段产物**——本阶段只增不改（除非该阶段明确说要重构）。
