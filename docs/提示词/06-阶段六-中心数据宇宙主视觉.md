# 06 · 阶段六：中心数据宇宙主视觉

## 本阶段目标

做出大屏的**镇屏之图**——中心区域「数据宇宙」主视觉：一个醒目的中心节点「DeepDVL 数据中枢」放射状连接 6-8 个业务节点，带涟漪脉冲与流光连接线。本阶段结束时，大屏有了视觉中心与「镇屏感」。

## 前置依赖

- 阶段五已完成：通用组件就绪，`BasePanel` 可用。
- 阶段三的 `hubNodes` 数据已就绪（8 个节点环绕中心，0-100 抽象坐标）。

## 为什么这么做

**大屏需要一个「视觉锚点」**。一屏七八个图表，如果没有一个中心主视觉，视觉就会散——观众不知道往哪看。中心图的作用是「把视线收拢」，同时承担「总览」语义。

**为什么不用地图**：地图是新手做中心图的第一反应，但它有三个坑：
1. 需要 geojson 数据（china.json 几百 KB），引入重依赖。
2. 地图视觉容易抢戏，配色一不慎就违和。
3. 教学项目引地图，学生注意力会被「地图怎么画」带偏，偏离「数据大屏」主线。

**用 cartesian2d 抽象拓扑**更好：坐标系 0-100，中心节点放 [50,50]，周围节点环形分布，用 `lines`（连接线）+ `effectScatter`（脉冲节点）+ `scatter`（中心大节点）实现。轻、可控、视觉自由度高，且与「深空数据宇宙」主题完美契合——节点即星，连线即星轨。

**实现要点**：
- `lines` 系列做连接线，开 `effect.show` 流光，营造「数据流动」感
- `effectScatter` 做周围节点，`ripple` 涟漪脉冲
- `scatter` 做中心节点，大 symbol + 强发光
- 背景叠一两条 `lines` 星轨环形（polyline，超低透明度）做装饰
- `tooltip` 显示节点名/活跃度/状态说明

**教学要点**：理解 ECharts 的「**多 series 叠加**」——一张图不是单个 series，而是多个 series 层叠（背景星轨 + 连接线 + 节点），通过 `z` 控制层序。这是大屏主视觉的核心技法。

## 要落地的产物

- `src/charts/DataHubChart.vue`：接收 `data: HubNode[]`，用 cartesian2d 渲染中心枢纽图。严格沿用阶段四的封装范式（init/setOption/watch/observeResize/dispose）。`data-testid="data-hub-chart"`。
- 视觉规范：
  - 中心节点：「DeepDVL 数据中枢」，大 symbol，青蓝强发光，涟漪
  - 周围节点：按 `value` 定大小、按 `status` 定颜色（good/warning/danger）
  - 连接线：青蓝半透明，开流光 effect
  - 背景星轨：1-2 圈环形 polyline，超低透明度青蓝虚线
  - tooltip：深底青字，显示名称/活跃度/状态说明
- `src/views/DashboardView.vue`：布局改成三栏 Grid（左 25% / 中 1fr / 右 25%），中心放 `BasePanel` + `DataHubChart`，左右各放后续阶段的图表占位（本阶段左侧可放已有的 LineTrendChart，右侧先空占位或放 PieStatusChart 占位——但本阶段不实现新图表，右侧占位即可）。

## 思考题 / 进阶

1. 为什么中心节点用 `scatter` 而周围节点用 `effectScatter`？（提示：中心要稳，周围要动；effectScatter 的涟漪适合表达「活跃」）
2. 多个 series 的 `z` 值如果不设会发生什么？（提示：默认按声明顺序，但显式设 z 更可控，避免装饰压住数据）
3. 流光 `effect.show` 性能开销在哪？节点很多时该不该关？（提示：每条线都跑动画，连线过多会卡）
4. 进阶：给中心节点加一个「呼吸」效果（symbolSize 随时间正弦变化）——用 ECharts 的定时 setOption 或自定义 series 实现。
