# 05 · 阶段五：核心指标卡片与通用组件

## 本阶段目标

抽出大屏的**通用组件三件套**：`BasePanel`（HUD 面板外壳）、`MetricCard`（指标卡片）、`ScreenHeader`（顶部标题栏 + 实时时间）。本阶段结束时，页面顶部有了标题栏，下面有一排指标卡片，每个图表都有了统一的 HUD 面板外壳——大屏的「模块感」出来了。

## 前置依赖

- 阶段四已完成：折线图封装就绪，封装范式已建立。

## 为什么这么做

**大屏的视觉一致性来自「通用组件」**。一个典型大屏有 6-8 个面板，如果每个面板自己写边框、标题、内边距，必然各写各的、参差不齐。抽成 `BasePanel` 后，所有面板共享同一套 HUD 外壳，视觉统一、改一处全改。

**三个组件的职责切分**：
- `BasePanel`：**纯外壳**——标题 + 内容 slot + HUD 装饰（四角发光边框）。不关心内容是什么。这是「容器优先」思想。
- `MetricCard`：**指标专用**——大数字 + 单位 + 趋势箭头 + 状态色。不复用 BasePanel，因为指标卡的视觉语言（大数字居中）和面板（标题+图表）不同。
- `ScreenHeader`：**顶部专属**——标题居中 + 左右流光装饰 + 实时时间。大屏「门面」。

**为什么 ScreenHeader 要实时时间**：大屏常驻展示，时间能给观众「这是实时的」直觉反馈，是数据大屏的标配元素。

**教学要点**：理解「**容器组件 vs 展示组件**」——`BasePanel` 是容器（提供外壳与 slot），图表是展示（填进 slot）。这种分工让复用性最大化。

## 要落地的产物

- `src/components/BasePanel.vue`：props `{ title: string }`，结构 `<section><header><span class="mark"/><h2>{{title}}</h2><slot name="extra"/></header><div class="body"><slot/></div></section>`。深空 HUD 风格：面板底 `--panel`、青蓝描边 `--border`、四角发光直角（伪元素 `::before/::after`）、`box-shadow` 内外发光。装饰克制，不压内容。
- `src/components/MetricCard.vue`：props `{ metric: SummaryMetric }`。布局：label（`--muted` 小字）+ 大数字（`--text` 粗体）+ 单位 + 趋势（↑绿 ↓红，带 `trend` 百分比）。状态色：good/warning/danger 对应左侧色条或角标。`data-testid="metric-card"`。
- `src/components/ScreenHeader.vue`：标题「DeepDVL · 深度数界实验室」居中，左右流光线条装饰，右侧实时时间（`setInterval` 每秒更新，格式 `YYYY-MM-DD HH:mm:ss`）。`onBeforeUnmount` 清定时器。
- `src/utils/format.ts`：数字格式化工具（如 `formatNumber(128936) → "128,936"`、`formatTrend(12.6) → "+12.6%"`），供 MetricCard 用。
- `src/views/DashboardView.vue`：用 `ScreenHeader` + 一排 `MetricCard`（v-for summary）+ 把 `LineTrendChart` 包进 `BasePanel`。布局用 CSS Grid。

## 思考题 / 进阶

1. `BasePanel` 用 slot 而不是 props 传组件，好处是什么？（提示：内容可以是任意 Vue 片段，不必为每种内容定义 prop 类型）
2. `MetricCard` 为什么不复用 `BasePanel`？（提示：视觉语言不同，强行复用会要加一堆条件分支，违背单一职责）
3. `ScreenHeader` 的时间如果不清理定时器会怎样？（提示：组件卸载后定时器还在跑，内存泄漏 + 改 ref 报警告）
4. 进阶：给 `BasePanel` 加一个 `loading` 态——数据未到时显示骨架屏，想想状态应该由谁传进来。
