# 02 · 阶段二：大屏布局与自适应

## 本阶段目标

搭出大屏的**布局骨架**：一个 16:9 画布能在任意浏览器窗口里**不变形**地居中铺满，并铺好「深空数据宇宙」的全局背景。本阶段结束时，你有一个带网格星云背景、16:9 不变形的空画布——还没有内容，但「舞台」搭好了。

## 前置依赖

- 阶段一已完成：Vite + Vue 3 + TS 工程可运行，`npm run build` / `lint` 全绿。

## 为什么这么做

**大屏的头号问题是适配**。大屏通常按 1920×1080 设计，但用户屏幕千差万别（1366×768、2560×1440、超宽屏…）。如果直接 `width:100vw; height:100vh`，16:9 的设计稿在非 16:9 屏幕上会被**拉伸变形**——图表变扁、圆变椭圆。

**核心解法**：用一个「画布容器」，其宽高比恒为 16:9，尺寸取「宽高里更紧的那个」：

```
width:  min(100vw, 100vh * 16/9)
height: min(100vh, 100vw * 9/16)
```

- 屏幕比 16:9 更宽（如 21:9）→ 高度顶满，宽度按 16:9 算，左右留黑
- 屏幕比 16:9 更窄（如 4:3）→ 宽度顶满，高度按 9:16 算，上下留黑

这样画布永远 16:9，内部用 1920×1080 的设计坐标摆放，**绝不变形**。`BigScreenLayout` 组件就是干这个的。

**为什么现在只做骨架不做图表**：适配是地基，地基不稳后面所有图表都跟着歪。先确认 16:9 不变形、无滚动条，再往里填东西。

**教学要点**：`min()` / `max()` 这两个 CSS 函数是适配利器，比 JS 算尺寸简单且无闪烁。配合 `overflow: hidden` 确保不出现滚动条。

## 要落地的产物

- `src/layouts/BigScreenLayout.vue`：16:9 画布容器，接收默认 slot。
- `src/views/DashboardView.vue`：先用 `BigScreenLayout` 包一个占位（显示「布局骨架就绪」），后续阶段往里填内容。
- `src/app/App.vue`：改为渲染 `DashboardView`。
- `src/app/styles.css`：全局视觉基线——`:root` CSS 变量（见 [00-视觉基线](./00-项目总览与使用说明.md#三视觉基线深空数据宇宙)）、body 深空背景（渐变 + 星云 radial-gradient + 网格）、`html/body/#app` 满屏 `overflow:hidden`。
- `index.html`：`<body>` 加 `class` 或确保 `#app` 满屏。

> 视觉基线色板与背景层叠写法严格按 [00-视觉基线](./00-项目总览与使用说明.md#三视觉基线深空数据宇宙) 来。

3. 网格背景为什么用 `mask-image` 淡出？不淡出会有什么问题？
